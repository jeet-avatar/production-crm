2. Backend API Endpoints

Method Endpoint Auth Purpose
POST /api/contracts/:id/generate-pdf JWT Generate PDF from contract template + rate card
POST /api/contracts/:id/send JWT Send signing link email to client
GET /api/contracts/sign/:token Public Get contract details for signing page
POST /api/contracts/sign/:token/request-otp Public Send 6-digit OTP to signer's email
POST /api/contracts/sign/:token/verify-otp Public Verify OTP, return session token
POST /api/contracts/sign/:token/submit OTP Session Submit signature (type/draw), capture audit data
POST /api/contracts/:id/countersign JWT Peter/Rajesh counter-signs in CRM
POST /api/contracts/:id/void JWT Void a sent/partially-signed contract
POST /api/contracts/:id/remind JWT Send reminder email to unsigned client

5. Security & Validation

✓ Included
  • Cryptographic signing token (crypto.randomBytes)
  • Token expires in 7 days
  • OTP: 6 digits, 10 min TTL, max 5 attempts
  • OTP codes hashed (bcrypt) in DB
  • Rate limit: 3 OTP requests per hour
  • SHA-256 document hash for tamper detection
  • IP + user agent captured at every step
  • Signed PDF stored in S3 (immutable)
  • All timestamps in UTC (ISO 8601)
  • HTTPS-only signing URLs
⚠ Not in Scope (v1)
  • Multi-signer (3+ parties)
  • Field-by-field placement (initials here, sign there)
  • Bulk send contracts
  • Contract versioning / redlining
  • Mobile app signing
  • UETA/ESIGN legal compliance notice (can add later)
